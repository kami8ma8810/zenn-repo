# X Clipper for Obsidian

X（Twitter）のポストを Obsidian に保存する Chrome 拡張機能。

## なぜ作ったのか

### X のブックマーク・いいねの問題点

X には「ブックマーク」や「いいね」機能がありますが、致命的な問題があります：

- **ポストが削除されたら遡れない** - 投稿者がポストを削除すると、ブックマークやいいねからも消えてしまいます。せっかく保存した有益な情報が永遠に失われます。
- **フォルダ分けはプレミアム会員限定** - ブックマークのフォルダ分け機能はプレミアム会員のみ。しかも、フォルダ分けしてもポストが削除されれば意味がありません。

### 解決策：Obsidian に保存

この拡張機能を使えば：

- **ポストの内容を Markdown で永続保存** - 削除されても手元に残る
- **画像も一緒に保存** - 添付画像もローカルにダウンロード
- **自分だけのナレッジベース** - Obsidian のタグやリンク機能で自由に整理
- **オフラインでも閲覧可能** - インターネット接続不要

## 機能

- ワンクリックで X のポストを Obsidian に保存
- 画像の自動ダウンロード・保存
- 引用ポストの内容も含めて保存
- スレッド（連投リプ）の保存（1つのノートにまとめるか、個別に保存するか選択可能）
- タグ付け機能
- 多言語対応（日本語 / 英語）

## インストール

### 必要なもの

1. **Obsidian** - https://obsidian.md/
2. **Obsidian Local REST API プラグイン** - Obsidian のコミュニティプラグインからインストール

### Obsidian 側の設定

1. Obsidian で「設定」→「コミュニティプラグイン」を開く
2. 「Local REST API」を検索してインストール
3. プラグインを有効化
4. プラグイン設定で HTTP サーバーを有効化（ポート: 27124）
5. API キーを設定（推奨）

### Chrome 拡張機能のインストール

※ 現在は開発版のみ。Chrome Web Store への公開は準備中です。

1. このリポジトリをクローン
2. `npm install` で依存関係をインストール
3. `npm run build` でビルド
4. Chrome で `chrome://extensions` を開く
5. 「デベロッパーモード」を有効化
6. 「パッケージ化されていない拡張機能を読み込む」をクリック
7. `dist` フォルダを選択

## 使い方

1. X（Twitter）でポストを開く
2. Chrome ツールバーの X Clipper アイコンをクリック
3. 必要に応じてタグを追加
4. 「保存」ボタンをクリック

## プライバシーとセキュリティ

- **データは一切収集しません** - すべてのデータはあなたのデバイスにローカル保存されます
- **外部サーバーへの送信なし** - ローカルの Obsidian との通信のみ
- 詳細: [プライバシーポリシー](./docs/PRIVACY_POLICY.md)
